Font-face

  1. These are types of font formats:
    1. svg
    2. eot
    3. otf
    4. woof2
  2. WOOF2 is the best font file formats to use in all browsers.
  3. The difference between:
    1. @font-face: must define a name for the font and use font properties.
    2. @keyframes/@mediaqueries/@supportsblocks: apply different styles.
  4. Some fonts come with licencing restrictions. That means they can be embendded in your publication and printed anywhere.
  5. We need to provide six font-face blocks if we want tosupport all the chosen fonts.
  6. It is roughly 160KB.
  7. We would use use --webkit- if we chose to offer WOOF2 as well as WOOF
  8. We can use local() to make sure the user's browser does not download the fonts they might already have.
  9. Sometimes we have to assign fallback fonts for our custom webfonts we add via @font-face. These are some thing to keep in mind when it comes to selecting these fallback fonts:
    1. try to load the fonts and the font content.
    2. set up font request.
  10. The browser can use one of several strategies for loading the fonts, e.g. showing nothing until the font is downloaded, or using a fallback font first. We can use font-display to help the browser decide on which strategy to use.
