CSS: Table-styles

Default values of the following properties:
  • border-spacing: 2px
  • table-layout: auto;
  • border-collaps: separate;
  • caption-side: top;
  • empty-cell: show;

Simulating HTML tables

HTML elements correspond to those values:
  • table: table
  • table-row-group: tbody
  • table-header-group: thead
  • table-footer-group: tfoot
  • table-row: tr
  • table-cell: td
  • table-column-group: colgroup
  • table-column: col
  • table-caption: caption

Further research

  1. The other declaration border-spacing requires before it can work effectively is border-collaps: separate.
  2. The downsides of using the various table* values for display on a set of elements, thereby simulating a table layout on non-table elements are:
    1. You will miss the relationship.
    2. It makes content inaccessible.
    3. It is difficult to control.
  3. The difference between table-layout: fixed and table-layout: auto
    1. table-layout: fixed: all the columns have the same width.
    2. table-layout: auto: makes the width of the column to fit the content.
  4. We can use border-spacing and other table-like property on non-table element.
  5. empty-cells requires border-collaps with a value separate before it can work effectively.
